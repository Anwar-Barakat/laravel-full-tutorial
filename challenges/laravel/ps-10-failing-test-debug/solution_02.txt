// ============================================================
// WHY Mail::fake() WORKS WITH OBSERVERS
// ============================================================

// Mail::fake() replaces the real mailer with a fake implementation before the test runs
// When Booking::factory()->create() fires, it:
//   1. Inserts the record into the database
//   2. Fires the 'created' Eloquent event
//   3. Laravel dispatches this to the BookingObserver::created() method
//   4. The observer calls Mail::to()->send(new BookingConfirmationMail(...))
//   5. The fake mailer intercepts the send call and records it
//   6. Mail::assertSent() checks the fake mailer's recorded dispatches
//
// If the observer is not registered, step 3 never happens
// If the observer is registered but the wrong method name is used (e.g., 'saving' vs 'created'),
// the mail will not fire at the right time
